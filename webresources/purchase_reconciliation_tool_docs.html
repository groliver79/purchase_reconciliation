📋 About This App

The Purchase Reconciliation Tool is a Microsoft Power Apps Model-Driven application built on Dataverse, designed to manage AJV-C purchase requests. It enforces business rules through C# server-side plugins and provides guided user input through internal LCNC business.

Audience This document is intended for developers and system administrators responsible for maintaining, extending, or deploying this solution. End-user guidance is covered in the separate User Guide.

🏗 Architecture

The solution follows a standard Power Platform layered pattern: Dataverse provides the data platform, server-side plugins enforce data integrity, and business rules handle client-side UX logic on UCI model-driven forms.

CreatePcaRecordPlugin.cs

Plugin biteam_purchaserequest Automatically creates a linked cr6c5_PCA record whenever a new biteam_purchaserequest record is created. Derives the PCA number by replacing TPC- with PCA- to the cr6c5_pcanumber field on Create, preventing duplicate prefixes if a record is already present.

Adding More Plugins A subsequent plugin class can be added to the same DLL without reregistering the assembly — only a new Step registration is needed. If a fourth plugin is added, consider refactoring to a single configurable plugin using the Unsecure Configuration field in the Plugin Registration Tool.

Deployment Steps

  1. Build the solution in Visual Studio (Release mode).
  2. Open the Plugin Registration Tool and connect to the target environment.
  3. Update the assembly (Upload) if already registered, or register new.
  4. Register or verify Steps for Create and Update on biteam_purchaserequest.
  5. Test by creating a new Purchase Request and confirming the TPC- prefix appears.

🚀 Deployment

All components are bundled in the managed solution. For environment promotion:

  1. Export the solution as Managed from the source environment.
  2. Import into the target environment via the Power Platform admin centre or CLI.
  3. Verify plugin assembly steps are active post-import.
  4. Run an end-to-end test: create a Purchase Request and validate prefix and field formatting.
